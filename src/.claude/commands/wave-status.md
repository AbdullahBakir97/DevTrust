# /wave-status

When the user runs this, do the following in order:

1. Read each file in `waves/` (wave-1 through wave-4).
2. For each wave, summarize: months covered, products it ships, current status (planning / building / shipped), and the single most important next action.
3. Render the result as a small markdown table:

   | Wave | Months | Products | Status | Next action |
   |------|--------|----------|--------|-------------|

4. Below the table, surface any wave that has slipped or has unresolved blockers.
5. End with one sentence — what should the user focus on this week?

Be concise. No headers. No emojis. Numbers where possible.
